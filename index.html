ClinixPRO - Hospital Management System

Overview

ClinixPRO is a comprehensive Hospital Management System designed to streamline and enhance the workflow within medical environments. Built on Node.js with a Nest.js backend and a Next.js frontend, it caters to four primary user roles: Super-admin, Doctor, Patient, and Receptionist.

Key Features

Super-admin
  • User Management: Create, Read, Update, and Delete functionalities for Doctors, Patients, and Receptionists.
  • Appointment Management: Access and view appointments across the system.
Doctor
  • Appointment Dashboard: Categorized lists for Today's, Pending, Upcoming, Completed, and Canceled appointments.
  • Patient & Receptionist Management: Access to patient lists and own receptionist lists.
  • Prescription & Invoice Control: CRUD operations for prescriptions and viewing capabilities for invoices.
Patient
  • Appointment Overview: View appointments with doctors and manage cancellations for pending appointments.
  • Doctor Selection: Access to lists of available doctors for appointments.
  • Prescription & Invoice Viewing: Access to personal prescriptions and view invoices.
Receptionist
  • Appointment Management: View doctor's appointments in categorized lists.
  • Patient & Doctor Interaction: Access to patient lists and doctor's prescription details.
  • Prescription Management: CRUD operations for prescriptions associated with assigned doctors.

Role-Based Functionalities

  • Super-admin: Complete control over user management and system-wide appointment viewing.
  • Doctor: Detailed appointment management and control over patient interactions, prescriptions, and invoices.
  • Patient: Access to personal appointment details, doctor selection, prescriptions, and invoice viewing.
  • Receptionist: Focused on managing doctor's appointments, patient interactions, and doctor-related prescriptions.
Purpose

ClinixPRO aims to streamline operations within medical facilities, providing an intuitive interface for efficient appointment scheduling, patient management, prescription handling, and invoice tracking. It enhances communication and accessibility among doctors, patients, and receptionists, ultimately optimizing the healthcare workflow.

Installation Steps

pnpm Package Management

  1. Check PNPM Installation: 
    pnpm --version
  2. If PNPM is not installed: 
    npm install -g pnpm

Server (Nest.js)

  1. Change Directory to Server: 
    cd server
  2. Setup .env file:

    Change .env.example file to .env

    Full field the .env file

  3. Install Dependencies and Start Server: 
    pnpm install
    pnpm start:dev

Clinic Software - Next.js Apps

  1. Doctor App: 
    cd doctor
    pnpm install
    pnpm dev
  2. Patient App: 
    cd patient
    pnpm install
    pnpm dev
  3. Receptionist App: 
    cd receptionist
    pnpm install
    pnpm dev
  4. Super Admin App: 
    cd super-admin
    pnpm install
    pnpm dev

Reset Password - React Apps

Configuring the App

  1. Open Project In Xcode
    • First make sure that you have installed the last version of X-code
    • Launch Xcode and navigate to the project folder. Open the .xcodeproj file to load your project into Xcode's workspace.
  1. Change App Name
    • Click on project navigator in general tab -> Disply Name -> (Enter New Name)
  1. Change App Identifier
    • Click on project navigator in signing & capabilities tab -> Signin -> (Enter New Bundle Identifier)
  1. Change BaseURL
    • Click on Model Folder -> Open APIManager.swift File -> Replace Your URL with baseUrl Parameter
  • Perfect Now Build The Project And Run On Physical Device Or Ios-Simulator
